### @ComponentScan & @Component
- @Component가 붙은 클래스를 @ComponentScan이 찾아서 자동으로 스프링 빈으로 등록하고 의존성 주입을 해줌
- 이때 의존성 주입이 필요한 생성자(현재까지는)에 @Autowired를 붙이전 자동으로 의존성 주입을 해줌
- 참고:
    - @Configuration에도 @Component가 들어가면 붙어있음
    - 여기서는 설정정보를 사용할 때 자동으로 스프링 빈으로 등록이 되서 있는 것임
        - 이거는 @Configuration은 제외했는데도 왜 스프링 빈으로 Config가 등록되었는지에 대한 답

### 동작 방식
1. @ComponentScan: 컴포넌트 스캔을 시작해서 @Component가 붙은 모든 클래스를 스프링 빈으로 등록
    - 이름 규칙
        - 빈 이름 기본 전략: 클래스 이름을 앞글자를 소문자로 만들어서 등록
        - 빈 이름 직접 지정: @Component("customName")

2. @Autowired 의존관계 자동 주입
- 생성자에 @Autowired를 지정하면 해당 스프링 빈을 찾아서 등록
- 기본 조회 전략은 "타입이 같은 빈을 찾아서 주입"

### AutoAppConfig 동작 방식
1. 설정 파일이므로 스프링 빈으로 등록해서 CGLIB 기술로 만듦
2. @ComponentScan이 붙어있으므로 컴포넌트 스캔 시작
3. 자동으로 빈 등록
4. 이후 @Configuration이므로 @Bean이 있다면 수동 빈 등록 시작
- 만약 @Configuration이 없다면
    - 지금은 문제가 발생하지 않음
    - AppConfig처럼 수동 등록 빈을 할 때 문제가 발생할 수 있음

### 탐색 위치와 기본 스캔 대상
- basePackage = "hello.core"
    - hello.core 패키지부터 조회를 시작

- 기본 전략: @ComponentScan이 붙은 설정 정보 클래스의 패키지 위치와 그 하위 패키지를 전부 탐색
- **가능한 프로젝트 최상단에 두자!! - 스프링 부트 기본 전략**

### 컴포넌트 스캔 기본 대상
- @Component
- @Controller: 스프링 MVC 컨트롤러로 인식
- @Service: 별다른 기능 없음
- @Repository: 스프링 데이터 접근 계층으로 인식, 데이터 계층의 예외를 스프링 예외로 변환
- @Configuration: 설정 정보로 인식, 스프링 빈이 싱글톤을 유지하도록 추가 처리를 해줌

> 주의: 어노테이션은 사실 상속 관계가 없음 -> 스프링이 지원해주는 기능!!

### 필터
- includeFilters: 컴포넌트 스캔 대상을 추가로 지정
- excludeFilters: 컴포넌트 스캔에서 제외할 대상을 지정

- 코드를 모면 BeanA에 @Component를 붙이지 않았는데도 컴포넌트 스캔을 해서 빈으로 등록함

### FilterType 옵션
- @Filter(type = FilterType._)
    - ANNOTATION: 기본값, 애노테이션을 인식해서 동작
    - ASSIGNABLE_TYPE: 지정한 타입과 자식 타입을 인식해서 동작 => 특정 클래스를 제외할 수 있음(스캔 대상이여도!)
    - ASPECTJ: AspectJ 패턴 사용
    - REGEX: 정규 표현식
    - CUSTOM: TypeFilter라는 인터페이스를 구현해서 처리

- 기본적으로 빈으로 등록할 대상에 @Component를 붙이기 때문에 includeFilters는 잘 사용하지 않음
- **그리고 항상 스프링 기본 설정에 최대한 맞춰서 사용하자!!**

### 중복 등록과 충돌(빈 이름 동일)
- 자동 빈 등록 vs 자동 빈 등록
    - 예외 발생

- 자동 빈 등록 vs 수동 빈 등록
    - 직접 스프링 컨테이너를 만들어서 할 경우, 수동 등록이 우선권을 가짐
        - 대원칙: 구체적인 것(명시적인 것)이 우선권을 가짐
    - 스프링 부트는 이것도 안되게 막아놓음
        - 설정을 통해 변경할 순 있음
        - spring.main.allow-bean-definition-overriding=true
