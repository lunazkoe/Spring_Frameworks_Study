## 스프링 컨테이너의 생성 과정
1. 스프링 컨테이너 생성
    - 스프링 컨테이너 생성(AppConfig.class)
    - 구성 정보 활용
2. 스프링 빈 등록
    - 빈 이름은 메서드 이름을 사용
    - 빈 이름을 직접 부여할 수도 있음
        - @Bean(name = "memberServiceCustomName")
3. 스프링 빈 의존관계 설정 - 준비
4. 스프링 빈 의존관계 설정 - 완료

### 참고 
- 스프링은 빈을 생성하고, 의존관계를 주입하는 단계가 나누어져있음
- 근데 자바 코드로 스프링 빈을 등록하면 생성자를 호출하면서 의존관계 주입도 한 번에 처리됨

### Bean 조회
- ac.getBean(이름)
- ac.getBean(이름, 타입)
- ac.getBean(타입)
- 이름은 중복되지 않게 하자!!
- 상위타입으로 가급적 조회하자!!
- 동일한 타입이 여러 개 있으면 이름으로 구분해서 조회하자!!

### BeanFactory와 ApplicationContext
- 둘 다 인터페이스이고 이걸 스프링 컨테이너라고 부름
- BeanFactory보다 ApplicationContext에서 부가적인 기능을 더 사용할 수 있음
- 구현체를 통해서 생성함 (AnnotationConfigApplicationContext(AppConfig.class))

### BeanDefinition
- 스프링 빈의 메타 데이터 정보
- 스프링 컨테이너는 이 정보만 보면 됨
    - 즉, XML이든 자바 코드이든 간에 ___Reader를 통해서 BeanDefinition만 생성해주면 됨
